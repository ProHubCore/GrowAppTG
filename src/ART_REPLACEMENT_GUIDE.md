# Замена изображений

Основные пути собраны в `core/assets/assetCatalog.js`.

Самый безопасный способ: оставить имя и путь файла прежними, а PNG просто заменить в `public/assets`. Тогда код менять не нужно.

## Рекомендуемая структура public/assets

```text
assets/
  backgrounds/
  buildings/
  characters/
  pots/
  plants/<plant-id>/stage-1.png ... stage-3.png
  seeds/
  items/
  tools/
```

## Размеры

- растения: прозрачный PNG, одинаковый холст для трёх стадий;
- карточки семян: квадратный прозрачный PNG;
- здания района: прозрачный PNG;
- фон: вертикальный, безопасный для обрезки на разных телефонах.

Для новой культуры добавляй запись в:
1. `features/plantation/data/seeds.js`;
2. `features/plantation/data/plants.js`;
3. `features/shop/shopItems.js`;
4. при необходимости — путь в `core/assets/assetCatalog.js`.

## Адаптивность и якоря PNG

Игра использует единую логическую сцену **390×844**. Она целиком масштабируется
под доступную область телефона. Не используйте `100vw`, `100vh` или координаты
от окна браузера для объектов внутри игры.

Правильно:

```css
.my-object {
  position: absolute;
  left: 50%;
  bottom: 120px;
  width: 180px;
  transform: translateX(-50%);
}
```

Неправильно:

```css
.my-object {
  left: 40vw;
  bottom: 18vh;
}
```

Для растения и ведра используйте один общий родительский контейнер. Нижняя точка
PNG растения должна быть прозрачным краем у основания стебля. Тогда стадии роста
останутся привязаны к земле и не будут прыгать при замене изображения.

Рекомендуемые правила для PNG:

- прозрачный холст одинакового размера у всех стадий одного растения;
- основание растения находится в одной и той же точке холста;
- `object-fit: contain`;
- не добавлять разные внешние пустые поля у стадий;
- декор привязывать к краям `.game-stage`, а не к экрану устройства.
