# Финальная адаптивная схема GrowApp

Игра использует гибридную архитектуру, как мобильные игровые приложения:

1. **Fullscreen-фон** находится на `.game-screen` и всегда заполняет реальный экран через `background-size: cover`.
2. **Игровой мир** имеет логические координаты `390 × 844` и масштабируется целиком в cover-режиме.
3. **Критичный HUD** получает вычисленные CSS-переменные видимой части сцены:
   - `--visible-left`
   - `--visible-right`
   - `--visible-top`
   - `--visible-bottom`
4. Декоративные края могут обрезаться. Кнопки, кошелёк, меню и стрелки должны использовать HUD-переменные и оставаться внутри экрана.
5. Telegram safe area добавляется только к верхнему HUD, а не уменьшает весь игровой мир.

## Добавление нового игрового объекта

Объекты мира: позиционировать внутри сцены 390 × 844.

```css
.new-world-object {
  position: absolute;
  left: 50%;
  bottom: 180px;
  transform: translateX(-50%);
}
```

HUD-кнопки: привязывать к видимой области.

```css
.new-hud-button {
  position: absolute;
  top: calc(var(--visible-top, 0px) + 16px);
  left: calc(var(--visible-left, 0px) + 16px);
}
```

## PNG растений

Все стадии одного растения должны иметь одинаковый прозрачный холст и одну точку основания. Рекомендуемый холст: `512 × 512`, основание: `x=256`, `y=490`.

## Что нельзя делать

- Не использовать `100vw` и `100vh` для объектов внутри игрового мира.
- Не дублировать основной фон внутри `.game-stage`.
- Не ставить важные кнопки строго в `left: 0` или `right: 0` — на cover-экранах этот край может быть обрезан.
